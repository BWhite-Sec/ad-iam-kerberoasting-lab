# Configs

- **`inputs.conf`** — Universal Forwarder input configuration used on `DC01`
  (sanitized; no IPs or hostnames beyond the lab's own).

  ## Prerequisite: enable Kerberos Service Ticket auditing

  By default, Windows Server does not log Event ID 4769 verbosely enough for
  this detection to work. Enable it on the domain controller before
  simulating any attack activity:

  ```powershell
  auditpol /set /subcategory:"Kerberos Service Ticket Operations" /success:enable
  ```

  Verify events are being generated locally before checking Splunk:

  ```powershell
  Get-WinEvent -FilterHashtable @{LogName='Security'; Id=4769} -MaxEvents 5 | Format-List TimeCreated, Message
  ```

  ## Notes on the Universal Forwarder

  Make sure the Universal Forwarder is installed **on the domain controller
  itself** (`DC01`), not on the domain-joined workstation. Event ID 4769 is
  generated by the Key Distribution Center (KDC) role, which only runs on
  domain controllers — a workstation's own Security log will never contain
  this event, regardless of forwarder health.
